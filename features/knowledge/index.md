# 大模型生成全流程 — 总分总教学

> 本文档用"总 → 分 → 总"的结构，帮助零基础读者理解一个语言模型从输入文字到输出文字的完整过程。所有概念都对应到 nanoGPT 的实际代码。

---

## 总：一句话概括大模型在干什么

**大模型就是一个"文字接龙"机器：给它一段话，它猜下一个字，然后把猜出来的字贴上去，继续猜下一个字，如此循环。**

比如你给它 `"从前有座"`，它猜 `"山"`，变成 `"从前有座山"`，再猜 `"，"`，变成 `"从前有座山，"` …… 一直猜下去。

这看似简单，但要做得好，需要解决几个问题：

| 问题 | 对应的模型部件 |
|------|--------------|
| 文字怎么变成模型能理解的数字？ | Token Embedding + Position Embedding |
| 模型怎么"理解"上下文？ | Multi-Head Self-Attention |
| 模型怎么"思考"和"推理"？ | Transformer Block × N 层 |
| 模型怎么输出一个字？ | LM Head + 概率采样 |
| 怎么让它越猜越准？ | 训练（损失函数 + 优化器） |

下面逐一拆解。

---

## 分：逐步拆解六个核心环节

### 环节 1：文字 → 数字 — 分词与嵌入

**生活中的类比**：把一句话翻译成密码本上的编号。

模型不能直接读文字。第一步是把文字切成小段（叫 **token**），每段对应一个编号。比如 `"你好世界"` 可能被切成 `["你", "好", "世", "界"]`，编号分别是 `[567, 891, 234, 456]`。

拿到编号后，模型用 **Embedding**（嵌入层）把每个编号变成一串小数（向量）。你可以理解为：每个字被翻译成了一个"含义坐标"。

```
"你好世界" → [567, 891, 234, 456] → [[0.12, -0.34, ...], [0.87, 0.21, ...], ...]
                                        每个 token 变成 n_embd 维向量
```

但光知道"每个字是什么"还不够，还得知道"每个字在第几个位置"。所以还有一个 **Position Embedding**，给每个位置也编一个向量。两个字相加，就得到了既包含"含义"又包含"位置"的输入。

**代码对应**（`model.py:127-129, 177-179`）：

- `wte = nn.Embedding(vocab_size, n_embd)` — 字 → 含义向量
- `wpe = nn.Embedding(block_size, n_embd)` — 位置 → 位置向量
- `tok_emb + pos_emb` — 相加得到最终输入

**为什么要取整到 50304？** `vocab_size=50304` 是 64 的倍数，GPU 做矩阵乘法时对齐更高效。

---

### 环节 2：理解上下文 — 多头自注意力（Multi-Head Self-Attention）

**生活中的类比**：读侦探小说时，你会不断回看前面的线索来理解当前情节。注意力机制就是让模型学会"回看哪些地方最相关"。

#### 什么是"注意力"？

对于句子中的每个字，模型会生成三个东西：

| 名称 | 类比 | 作用 |
|------|------|------|
| **Query (Q)** | "我在找什么线索？" | 当前位置发出的"查询" |
| **Key (K)** | "我有什么线索？" | 每个位置提供的"标签" |
| **Value (V)** | "我的具体内容" | 每个位置的实际信息 |

模型拿 Q 去和所有位置的 K 做比较（点积），越相关得分越高，然后按得分加权汇总所有位置的 V。这就是"注意力"——**关注哪些位置、忽略哪些位置**。

#### "多头"是什么意思？

把注意力分成多组（头），每组独立计算，关注不同角度。就像读书时有的注意力放在人物关系，有的放在时间线，有的放在环境描写。最后把所有头的结论拼在一起。

#### "因果"是什么意思？

生成第 5 个字时，只能看到第 1-4 个字，不能"偷看"第 6 个字。用下三角掩码实现。

**代码对应**（`model.py:29-76`）：

```
Q, K, V = c_attn(x).split(n_embd)     # 一次投影，拆成三份
att = softmax(Q @ K^T / sqrt(d_k))     # 算注意力分数
y = att @ V                             # 加权汇总
y = c_proj(y)                           # 输出投影
```

**Flash Attention**（`model.py:45,62-64`）：PyTorch ≥ 2.0 时自动启用 GPU 加速，不用真的存下那个巨大的注意力矩阵，既省内存又更快。

### 环节 3：思考与推理 — Transformer Block

**生活中的类比**：注意力帮你看清线索，但光看清还不够，还得"想一想"。这个"想"就是前馈网络（MLP）。

一个 Transformer Block 的结构就像**两步思考**：

```
第一步：看清上下文
  x = x + Attention(LayerNorm(x))

第二步：深入思考
  x = x + MLP(LayerNorm(x))
```

#### Pre-Norm（先标准化再处理）

每一步都是先做 LayerNorm（标准化，让数值分布稳定），再做 Attention 或 MLP。这叫 **Pre-Norm**，比先处理后标准化（Post-Norm）训练更稳定。

#### 残差连接（x = x + ...）

每步的输出是"原来的 x 加上新处理的结果"。这像抄作业时保留自己的答案，只在旁边补充新想法。好处是：即使某一层没学好，信息也能直接"跳过"传下去，不会丢失。

#### MLP（前馈网络）

```
Linear(n_embd → 4×n_embd) → GELU → Linear(4×n_embd → n_embd)
```

先把信息展开到 4 倍宽度（"展开思路"），过 GELU 激活函数（"非线性思考"），再压缩回来。GELU 比 ReLU 更平滑，是 GPT 和 BERT 的标配。

#### 为什么要叠很多层？

一层 Block 能学到的模式有限。叠 12 层（或更多），每层在上层的基础上继续提炼，就能理解越来越复杂的语义关系。

**代码对应**（`model.py:94-106, 130, 180-181`）：

```python
# 12 层 Block 堆叠
h = nn.ModuleList([Block(config) for _ in range(n_layer)])

# 前向传播时依次通过
for block in h:
    x = block(x)
```

### 环节 4：输出预测 — LM Head 与概率采样

**生活中的类比**：作文老师给你一个词表，让你圈出最可能接在后面的那个词。

#### LM Head

经过 N 层 Transformer Block 后，每个位置都有一个向量表示。最后用一个线性层（**LM Head**）把这个向量映射回词表大小的一组分数（logits）。

```
隐藏向量 (n_embd) → LM Head → logits (vocab_size)
```

每个词表中的 token 都有一个分数，分数越高表示模型认为它越可能是下一个字。

#### Softmax — 把分数变成概率

用 softmax 函数把 logits 转换成概率（所有概率加起来等于 1）。

#### Temperature — 控制随机性

```
logits = logits / temperature
```

- `temperature` 很小（如 0.1）：概率集中在最高分的 token，输出很确定、很无聊
- `temperature` 很大（如 2.0）：概率趋于平均，输出很随机、很有创意
- `temperature = 1.0`：不调整，保持原始分布

#### Top-K — 只从最好的 K 个里选

先只保留分数最高的 K 个 token，把其他的概率清零，再从剩下的里采样。避免采到离谱的低概率词。

**代码对应**（`model.py:306-329`）：

```python
logits = logits[:, -1, :] / temperature    # 取最后位置，除以温度
if top_k is not None:
    # 只保留 top-k
    v, _ = torch.topk(logits, top_k)
    logits[logits < v[:, [-1]]] = -float('Inf')
probs = F.softmax(logits, dim=-1)           # 变概率
idx_next = torch.multinomial(probs, 1)      # 掷骰子选一个
idx = torch.cat((idx, idx_next), dim=1)     # 贴上去，继续
```

#### 自回归 — 循环往复

生成一个 token 后，把它追加到输入末尾，再喂给模型生成下一个。直到达到最大长度。

---

### 环节 5：权重共享 — 省参数的巧思

**生活中的类比**：一个字典既用来查字义（输入），也用来判断该写什么字（输出），一本字典两个用途。

`lm_head`（输出层）的权重和 `wte`（输入嵌入层）是同一份参数：

```python
self.transformer.wte.weight = self.lm_head.weight  # model.py:138
```

好处：
- **省参数**：不用多存一份 vocab_size × n_embd 的矩阵
- **语义一致**：输入和输出在同一个向量空间里，模型更容易学好

---

### 环节 6：训练 — 让模型越猜越准

**生活中的类比**：做填空题，写错了老师帮你打分，你根据分数调整下次的答案。

#### 损失函数 — 给模型打分

用 **交叉熵损失**（Cross-Entropy Loss）衡量模型预测的概率分布和正确答案之间的差距。损失越小，猜得越准。

```python
loss = F.cross_entropy(logits.view(-1, vocab_size), targets.view(-1))
```

#### 优化器 — 根据分数调整参数

**AdamW** 优化器根据损失函数的梯度（方向）来调整模型的每个参数，让下次猜得更准。

**分组权重衰减**（`model.py:263-287`）：
- 权重矩阵（2D 参数）：施加 weight decay（正则化，防止过拟合）
- 偏置和 LayerNorm（1D 参数）：不施加，因为它们本身就是调节用的

#### 权重初始化 — 起跑线很重要

```
普通层：  N(0, 0.02)                    # 小随机数
残差投影：N(0, 0.02 / sqrt(2 * n_layer))  # 更小的随机数
```

为什么残差投影要更小？因为每层残差分支都会叠加信号，如果不缩小初始化，层数越多信号越强，会爆炸。

## 总：全景回顾

把所有环节串起来，大模型的完整工作流是：

```
你输入一句话
    │
    ▼
[1] 分词 + 嵌入：文字 → 编号 → 含义向量 + 位置向量
    │
    ▼
[2-3] N 层 Transformer Block，每层：
    │    先看上下文（注意力）→ 再深入思考（MLP）
    │    每步都保留原始信息（残差连接）
    │    每步都先标准化（Pre-Norm）
    │
    ▼
[4] 输出预测：隐藏状态 → LM Head → 分数 → 概率 → 采样一个字
    │
    ▼
[5] 把采样的字贴到输入末尾，回到 [1]，循环往复（自回归生成）
    │
    ▼
你得到完整的一段话
```

**训练时**：和上面类似，但不是循环生成，而是一次性输入完整句子，让模型预测每个位置下一个字是什么，算损失，更新参数。

**六个关键设计决策回顾**：

| 决策 | 为什么 |
|------|--------|
| Pre-Norm（先标准化） | 训练更稳定 |
| 残差连接 | 信息不丢失，梯度不消失 |
| 多头注意力 | 多角度理解上下文 |
| 因果掩码 | 只看过去，不看未来 |
| 权重共享（输入=输出） | 省参数，语义一致 |
| 缩放初始化 | 防止深层网络信号爆炸 |

理解了这些，你就理解了 GPT 类大模型的核心原理。
